# Start Here

DV Debug Lab is organized around real verification-debug problems.

Use it like this:

1. Search your symptom.
2. Open the matching article.
3. Compare the broken and fixed examples.
4. Follow the debug checklist.
5. Download or copy the example structure.
6. Adapt it to your testbench.

## Best path for beginners

1. [SystemVerilog](systemverilog/index.md)
2. [UVM Debug](uvm-debug/index.md)

4. [CDC & X-Propagation](cdc-xprop/index.md)

## How articles are structured

Each article follows the same template:

- Problem summary
- Where this happens
- Why it happens
- Minimal broken example
- Corrected example
- Debug flow
- Checklist
- Common simulator messages
- Interview angle
- Downloadable files
- Related topics

Repetition is not laziness here. It is navigation with manners.
